Cascata

A regra da cascata no CSS é o conjunto de regras que o navegador usa para decidir qual estilo aplicar quando múltiplas regras conflitantes se aplicam a um mesmo elemento. Ela é chamada de “cascata” porque o CSS segue uma ordem de prioridade em forma de "queda", determinando qual estilo vence.

Especificidade

  • Especificidade do seletor pela tag

    • É o seletor mais básico. Ele se refere diretamente a uma tag HTML, como p, div, h1, etc. Esse tipo de seletor tem o menor peso de todos. Pontuação de especificidade: 1

  • Especificidade do seletor pela classe

    • Refere-se a uma classe atribuída a um ou mais elementos HTML. É mais específico do que o seletor por tag, e é o tipo mais comum usado em projetos. Pontuação de especificidade: 10

  • Especificidade do seletor pelo id

    • Refere-se a um identificador único de um elemento HTML. É mais específico que os seletores por tag e por classe. Pontuação de especificidade: 100

Combinadores

  • Combinação de seletores com espaço:

    • Essa combinação significa que o seletor está buscando qualquer elemento h1 que esteja dentro de um elemento com a classe .primeiro, em qualquer nível de profundidade.

  • Combinação de seletores com >

    • Neste tipo de combinação, o CSS seleciona apenas o elemento h1 que for filho direto de um elemento com a classe .primeira.

  • Combinação de seletores com +

    • Esse seletor seleciona um h1 que vem logo depois de outro h1, dentro de .primeira. Ele se chama irmão adjacente, pois busca um elemento que esteja imediatamente após o anterior no mesmo nível.

Box Model

O Box Model (ou Modelo de Caixa) é a forma como o CSS enxerga e estrutura todos os elementos HTML na página. Cada elemento visual é tratado como uma caixa retangular, composta por várias camadas. Essas camadas determinam tamanho, espaçamento e comportamento do elemento na tela.

Margin

A margin é o espaço externo, ou seja, a distância entre o elemento e outros elementos ao seu redor. Serve para afastar elementos uns dos outros e organizar melhor o layout.

Padding

A propriedade padding define o espaço interno entre o conteúdo do elemento e sua borda. Ele "empurra" o conteúdo para dentro. É útil para dar respiro ao texto ou imagem dentro de uma caixa, sem alterar o espaço ao redor.

Border

A border é a linha que envolve o padding e o conteúdo. Pode ter cor, espessura e estilo. Ela pode ser usada tanto para destacar elementos visualmente quanto para delimitar áreas na página.

Shorthand

  • Com 1 valor: aplica o mesmo valor para todos os lados (top, right, bottom, left).
  • Com 2 valores: O primeiro valor é aplicado ao topo e base (top e bottom). O segundo valor é aplicado às laterais (right e left).
  • Com 3 valores: O primeiro valor é o top. O segundo valor é para left e right. O terceiro valor é o bottom.
  • Com 4 valores: Os valores são aplicados no sentido horário (top, right, bottom, left)

Display

display: block

Quando um elemento tem display: block, ele ocupa toda a largura disponível do contêiner pai, empurrando os outros elementos para baixo (em uma nova linha). Ele sempre começa em uma nova linha.

    Exemplos de elementos:

  • div
  • p
  • h1 até h6
  • section, article, nav, etc.

display: inline

Elementos com display: inline não quebram a linha. Eles ficam ao lado de outros elementos, um após o outro, como se fossem texto. Eles ocupam apenas o espaço necessário para seu conteúdo, sem forçar quebra de linha.

    Exemplos de elementos:

  • span
  • a
  • strong, em, b, i

Pseudo-Classes

  • :first-child : seleciona o primeiro filho de qualquer tipo
  • :last-child : seleciona o último filho de qualquer tipo
  • :nth-child(n) : seleciona o nº filho geral, contando todos os elementos
  • :nth-of-type(n) : seleciona o nº filho de um tipo específico de tag
